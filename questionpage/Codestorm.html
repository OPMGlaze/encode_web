Q1 You are tasked with solving a classic problem of combinatorial explosion under the constraints of infinite supply. Given a target integer amount N (representing cents), and a set S = {S₁, S₂, ..., Sₘ} of coin denominations (each an integer > 0), your objective is to compute the total number of distinct combinations of coins from set S that sum exactly to N. Constraints: You may assume an unbounded (infinite) supply of each coin denomination in S. The order of coin selection is irrelevant; combinations differing only in order are considered identical. Formally, determine the number of unique multisets of coins from S such that the sum of their values equals N. Examples: For N = 4 and S = {1, 2, 3}, the distinct combinations that satisfy the condition are: {1,1,1,1} {1,1,2} {2,2} {1,3} Output: 4 For N = 10 and S = {2, 5, 3, 6}, the valid combinations are: {2,2,2,2,2} {2,2,3,3} {2,2,6} {2,3,5} {5,5} Output: 5 Your task is to devise an efficient algorithm to compute the number of such unique combinations for arbitrary N and S. Naïve brute-force approaches will likely succumb to performance degradation for large inputs.
Q2 Aryan is playing an exciting adventure game where he must cross a series of magical stones arranged in a straight line. Each stone has a number written on it, which tells Aryan the maximum number of steps he can jump forward from that position. For example: If the stone shows 3, Aryan can jump to any stone between 1 to 3 steps ahead. If the stone shows 0, Aryan is stuck and cannot move forward from there. Aryan starts at the first stone and his goal is to reach the last stone using the minimum number of jumps possible. You are given an array arr[] of non-negative integers, where each element represents the maximum number of steps Aryan can jump forward from that stone. Your task is to write a Python function that returns the minimum number of jumps needed to reach the last stone starting from the first. If Aryan cannot reach the last stone, return -1. Number of stones: 2 ≤ arr.length ≤ 10⁴ Stone values: 0 ≤ arr[i] ≤ 10⁴ Write a Python function to solve the problem. Test the function on the given examples. Think about how you can optimize the solution for larger arrays (hint: greedy approach).
Q3 You are playing the classic "Bulls and Cows" game with a friend, where you secretly write down a number, and your friend tries to guess it. After each guess, you give them a hint based on two things: the number of bulls, which are digits that are correct in both value and position, and the number of cows, which are digits present in the secret number but in the wrong position. For example, if the secret is "1807" and the guess is "7810", then there is 1 bull ("8") and 3 cows ("7", "1", and "0"). Keep in mind that both the secret and guess may contain duplicate digits. Your task is to write a program or describe a method that compares the secret number and your friend’s guess, and returns a hint formatted as "xAyB" where x is the number of bulls and y is the number of cows.
Q4 You are working as a software engineer on a real-time monitoring system for a smart city infrastructure project. One component of the system is responsible for tracking temperature fluctuations recorded by sensors deployed across various locations. These sensors continuously send temperature readings as an array of integers. Due to resource constraints, you can only process the data in chunks — using a sliding window of fixed size k. Problem: As part of the system’s analytics, you need to continuously calculate the highest temperature reading in the last k time units as new data arrives. This is critical for real-time alerts (e.g., overheating in an area), so the solution must be efficient even for large datasets. Your Task: You are given: An array nums representing the sequence of temperature readings over time. An integer k representing the window size (e.g., how many recent readings to analyze at a time). Write a function that returns an array of the maximum temperature for each sliding window of size k. Each time the window moves one position to the right, compute and store the maximum of the current window.
